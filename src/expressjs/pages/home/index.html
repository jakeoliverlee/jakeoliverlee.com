Resume Website

  • Released: 03/05/2023
  • GitHub account v1.0.0

What problem did this solve?

  • Provides me with an online presence and allows potential employers to view my skills in a centralized location, so they can more efficiently assess whether I am a good fit for the role or not.
  • Enables me to showcase my proficiency in designing and developing fully functional websites, using HTML, CSS, and JavaScript to create engaging user interfaces, as well as integrating non-relational databases such as MongoDB to manage dynamic content.

Who are my user base?

  • Recruiters, hiring managers, clients as well as other potential employers.

What are some of the technical decisions I made?

  • The frontend for this project was written almost entirely in HTML and Tailwind CSS, with a small amount of Javascript. I needed this project to be fully mobile-responsive as well as supporting Dark Mode/Light Mode - TailwindCSS made this very simple to achieve.
  • The backend is written in NodeJS, based on the ExpressJS framework, which I used in my developer environment to spin up a locally hosted server.
  • Knowing that a single-page app (SPA) like this one didn't require a complex infrastructure, I opted for Google's PaaS service, Firebase. Firebase offers me a comprehensive set of tools, such as Google Analytics, to monitor my site's information.
  • For CI/CD, I utilized Github Actions, which executes tests on my code using Cypress end-to-end testing framework, installs necessary dependencies, and deploys my application to Firebase.

What was the outcome and what did I learn?

  • This project taught me a lot about UI/UX design and creating an engaging user experience, which was a new skill for me considering my background in backend-focused tech.
  • I learned about the simplicity of creating a CI/CD pipeline using Google's Firebase and Github Actions.
  • I learned how to test frontend code, specifically using Cypress for component and E2E testing.
  • Integrating dark and light mode compatibility was extremely simple with Tailwind, allowing my project to be accessible for those during the day as well as at night!

Streamline

  • Released: 30/04/2023
  • GitHub account Amazon Inc. v0.0.1

What problem did this solve?

  • Streamline is an internal platform built entirely in Python that data centre staff interact with to speed up various operational processes. Streamline can perform various tasks, the most popular being it's crunching of ticket data, which previously done manually would take around 15-20 mins each day. Streamline's ticket crunching algorithm meant that technicians no longer needed to manually count ticket numbers, Streamline does this calculation automatically by leveraging Python's pandas libary.
  • Streamline's release inspired me to create a community for data centre staff within AWS who have a passion for development. These individuals are natural problem-solvers and want to utilize their development skills to increase the efficiency of operating data centres. By doing so, they aim to provide the best experience for not only their data centre colleagues but also for AWS stakeholders and customers.

Who are my user base?

  • Data Centre staff in AWS London Data Centres

What are some of the technical decisions I made?

  • Streamline is a web application developed entirely in Python; based on the Streamlit web framework. Streamlit abstracts a lot of the complexities of the backend from me so I can focus on writing the Python code that makes up the application. Under the hood, streamlit uses WebSockets to enable real-time updates and low latency, Streamlit can also manage widget states on the frontend and syncs the states automatically to the backend.
  • Within Streamline, there is a ticket-data crunching service which takes in a CSV as input and returns sorted data in a table format as output. I am achieving this using Python's Pandas library. Streamline uses regex under the hood to check the validity of user input, user uploads csv, pandas operation is performed on the data in the CSV, data is outputted to the frontend in table format for the user to see.
  • Streamline is a fully-redundant, secure, and performant web application hosted entirely using AWS services. It is hosted on auto-scaled EC2 instances spread across three AZs and served by an Application Load Balancer (ALB). Streamline's unique domain name uses Route53 for fast and reliable DNS resolution, and CloudWatch for monitoring the EC2 instances.
  • For CI/CD, I am also using AWS services, namely CodePipeline, CodeBuild, and AWS's code repository, CodeCommit. I am using an S3 bucket to store a copy of my git repository which my ec2 instances read from when I make a commit to my main branch.
  • Streamline utilizes IaC through AWS CDK, which I wrote in TypeScript.

What was the outcome and what did I learn?

  • The outcome of the project was a fully functional platform for data centre staff to use, the feedback I have received from colleagues was all very positive and the project has been a success so far while Streamline continues to be an organic project.
  • Streamline saw a significant boost in DCO efficiency, Streamline's algorithm meant that ticket-data crunching now only took ~1 minute (which includes the time it takes to get the csv data and runtime), compared to the manual way which took ~15 minutes to complete; a significant increase in time-spent effiency of ~1400%.
  • I learned about the utility of TMUX and how it can boost developer producitivity.
  • Throughout my development of my CI/CD pipeline for Streamline, I picked up some familiarity with Typescript as well as some of the services AWS offers to build pipelines.
